# pCube Sharing Debugger

Local preview of how a URL appears when shared on social networks.
Extracts Open Graph, Twitter Cards, JSON-LD, and renders realistic previews for Facebook, Twitter/X, and LinkedIn.

Built for debugging sites that are only accessible from a whitelisted IP — no external API calls, everything runs locally.

## Features

- **Metadata extraction** — `og:*`, `twitter:*`, `meta description`, canonical URL, favicon, JSON-LD
- **Social preview cards** — realistic CSS replicas of Facebook, Twitter/X, and LinkedIn link previews
- **JS rendering** — optional Puppeteer/Chromium for SPAs with JS-injected meta tags
- **Screenshot** — viewport screenshot captured when JS rendering is enabled
- **Dark mode** — System / Light / Dark toggle, persisted in localStorage
- **Warnings** — detects missing or undersized images, missing titles, conflicting tags
- **Tailwind v4** — modern CSS pipeline with automatic dark mode support

## Quick start

```bash
npm install
npm start
```

Opens `http://127.0.0.1:3333` in the browser.

### Commands

| Command | Description |
|---------|-------------|
| `npm start` | Build CSS + start server on `127.0.0.1:3333` and open browser |
| `npm run dev` | Server with `--watch` + Tailwind watcher in parallel; CSS rebuilt automatically |
| `npm run build:css` | Rebuild Tailwind CSS (after editing `src/input.css`) |
| `PORT=4444 npm start` | Custom port |

The server listens on `127.0.0.1` only (no external exposure).

## API

### `POST /api/debug`

```json
{ "url": "https://example.com/page", "jsRender": false }
```

- `jsRender: true` — uses Puppeteer (headless Chromium). Useful for SPAs (React, Vue) that inject meta tags via JavaScript.
- `jsRender: false` (default) — direct HTTP fetch + cheerio parsing (fast).

**Response:**

```json
{
  "url": "https://example.com/page",
  "meta": { "og": {}, "twitter": {}, "general": {}, "jsonld": [], "all": [] },
  "previews": { "facebook": {}, "twitter": {}, "linkedin": {} },
  "warnings": [],
  "jsRender": false,
  "screenshotUrl": null
}
```

When `jsRender: true`, `screenshotUrl` contains a base64 JPEG of the page viewport.

## Architecture

```
├── server.mjs          Express entrypoint
├── src/
│   ├── scraper.mjs     Native HTTP fetch + cheerio HTML parsing
│   ├── puppeteer.mjs    Optional headless Chromium wrapper
│   ├── previews.mjs    Preview card data + warning generation
│   └── input.css       Tailwind v4 source (→ public/style.css)
├── public/
│   ├── index.html      Vanilla HTML UI
│   ├── app.js          Frontend logic
│   └── style.css       Auto-generated by Tailwind (do not edit)
├── Dockerfile           Multi-stage Docker build (with Chromium)
└── docker-compose.yml
```

## Docker

```bash
HOST=0.0.0.0 docker compose up -d --build
```

The container includes Chromium via apt, configured via `PUPPETEER_EXECUTABLE_PATH`. For external access, set `HOST=0.0.0.0` (default is `127.0.0.1`, local only).

For HTTPS, place behind a reverse proxy (Caddy, Nginx, Traefik).

## Notes

- `og:image` must be ≥ 200×200 px; ≥ 600 px recommended for best results
- `og:image` is the only image tag used by Facebook and LinkedIn; `twitter:image` is X/Twitter only
- If `og:title` is missing, the `<title>` HTML tag is used as fallback
- `puppeteer` downloads Chromium (~300 MB) on `npm install`. Remove it from `package.json` if not needed — JS rendering will show a clear error message
